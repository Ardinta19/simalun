# Internasionalisasi (i18n) di SIMALUN

## Konteks

SIMALUN adalah sistem internal Azka Laundry (UMKM Jambi). Audiens-nya
**100% berbahasa Indonesia** — tidak ada rencana multi-bahasa.

Tapi kami tetap menerapkan **infrastruktur i18n minimal** dengan dua
alasan:

1. **Konsolidasi string** — istilah yang muncul berulang di banyak view
   (mis. "Sedang Dicuci", "Lihat Semua", "Hapus") dipusatkan di file
   `lang/id/*.php`. Kalau Azka mau ganti istilah jadi "Proses Cuci",
   cukup edit satu file — bukan grep ke 80+ blade view.

2. **Sistem siap kalau perlu** — kalau besok ada cabang Azka di luar
   Jambi yang pakai dialek beda atau perlu English, struktur sudah ada.

## Struktur file lang

```
lang/
└── id/
    ├── validation.php   ← rule validation Laravel (existing)
    ├── orders.php       ← status, slot, zona, label form pesanan
    ├── finance.php      ← halaman keuangan & laporan
    ├── ui.php           ← tombol & label umum (Simpan, Batal, dll)
    └── messages.php     ← pesan flash success/error
```

## Cara pakai

### Di Blade view

```blade
<button>{{ __('ui.buttons.save') }}</button>

<span>{{ __('orders.status.'.$order->status) }}</span>

@if(session('success'))
    <div>{{ session('success') }}</div>
@endif
```

### Di PHP (controller / model)

```php
return back()->with('success', __('messages.success.saved'));

return back()->withErrors(['code' => __('messages.voucher.invalid')]);
```

### Dengan parameter

```php
__('messages.voucher.min_order', ['amount' => number_format(50000, 0, ',', '.')])
// "Minimum order Rp 50.000 untuk pakai voucher ini."
```

## Pola yang sudah dirapikan

| Lokasi | Pola |
|---|---|
| `Order::getStatusLabelAttribute()` | Pakai `trans('orders.status.'.$status)` |
| Validation rules | Sudah pakai `lang/id/validation.php` (Sprint 1) |

## Yang BELUM direfactor (intentional)

83 blade view masih banyak punya hardcoded string ("Pesanan", "Total",
"Lihat Semua", dll). **Tidak direfactor di PR ini** karena:

1. Risiko merge conflict besar (tiap perubahan UI/UX selanjutnya
   berpotensi konflik).
2. Untuk UMKM single-bahasa, value tambahan minimal vs effort.

**Iterasi mendatang**: tiap kali sentuh sebuah view (UI polish, fitur
baru, bug fix), sekalian replace hardcoded Indonesian string yang ada
di file itu pakai `__()`. Approach incremental ini lebih aman dan
konsisten dengan velocity tim.

## Kunci yang sudah disiapkan

Lihat 4 file di `lang/id/` untuk daftar lengkap. Highlight:

- **`orders.status.*`** — 8 status (menunggu, dijemput, dicuci,
  disetrika, siap, dikirim, selesai, dibatalkan)
- **`orders.pickup_time.*`** — pagi/siang/sore (versi panjang & short)
- **`orders.zone.*`** — A/B/C dengan range km
- **`orders.payment.*`** — cod/cash/transfer/qris
- **`ui.buttons.*`** — 17 tombol umum
- **`ui.states.*`** — active, inactive, loading, dst.
- **`messages.order.*`** — pesan flash terkait pesanan
- **`messages.voucher.*`** — pesan flash terkait voucher

## Konvensi penamaan kunci

- **lowercase + dot notation**: `ui.buttons.save`, bukan `UI_BUTTONS_SAVE`
- **Domain-first**: `orders.*`, `finance.*` — bukan `status.order`
- **Tidak ada string panjang** di kunci — string panjang langsung
  hardcode di view kalau cuma muncul sekali.
